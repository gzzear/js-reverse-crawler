# 字体混淆



### 字体混淆

> 什么是字体混淆

在常规网页中，浏览器渲染文本的逻辑是：

1. HTML 里的代码是 `1` (Unicode: `U+0031`)。
2. 浏览器去默认字体（如 Arial）里找 `U+0031` 对应的形状。
3. 屏幕显示“1”。

而在**字体混淆**的网页（如人人车、大众点评、猫眼电影）中，逻辑变成了这样：

1. **自定义字体**：网站动态下发一个 `.ttf` 或 `.woff` 文件。
2. **乱码映射**：HTML 里的代码是一个生僻字或私有区字符，比如 `` (Unicode: `U+E904`)。
3. **偷梁换柱**：在那个自定义字体文件中，厂家故意把 `U+E904` 这个编码对应的**字形（Glyph）**画成了数字 **“1”** 的样子。
4. **结果**：人类在浏览器看到的是“1”，但爬虫抓取的源码是 `` 或者 ``。

更恶心的是，**这个映射关系是动态的**。

- 刷新一下网页，HTML 里的代码可能变成了 `` (U+E905)。
- 同时字体文件也换了，新字体里 `U+E905` 画的才是“1”。

这就是为什么你写死字典（`{'': '1'}`）不管用的原因。

* 核心结构

当我们用 `TTFont` 打开一个字体文件时，最关注两个“表（Table）”：

1. **`cmap` (Character Map)**：字符映射表。
   - 作用：它告诉计算机，Unicode 编码 `0xE904` 对应的**字形名称（Glyph Name）**是什么（比如叫 `glyph00005`）。
2. **`glyf` (Glyph Data)**：字形数据表。
   - 作用：它存储了 `glyph00005` 具体是怎么画出来的（贝塞尔曲线的坐标点）。**这是破解的关键**。
